# 생기부 연계 상담 (세특 기반 심화 탐구) 기획 명세서

## 1. 기획 배경 및 목표
* **현황 및 문제점:** 기존 방식('아쉬웠던 점' 입력)은 대학에서 가장 선호하는 '전공 적합성 심화 트랙'을 구성하기에 다소 피상적이고 부정적인 뉘앙스를 줄 수 있음.
* **해결 방안:** 실제 학교 선생님이 작성해주신 **직전 학년/학기의 '세특 원문'**을 바탕으로 학생의 능력을 파악하고, 이번 학기에 새롭게 배울 **'타겟 과목의 실제 진도(단원명)'**와 융합하여 꼬리에 꼬리를 무는 연계 탐구 주제를 제안하는 초개인화 서비스를 구축함.

---

## 2. 사용자 입력 폼 구조 개편안 (WritePostPage)

사용자가 고민 없이 직관적으로 작성할 수 있도록 폼을 2개의 섹션으로 분리합니다.

### Section 1: ⏪ 나의 이전 활동 정보 (베이스캠프)
학생이 자신의 생기부를 열어놓고 참고하거나 복붙하는 영역입니다.

1. **활동 시기:** `[학년 선택 (1~3)]` `[학기 선택 (1~2)]` (Dropdown)
2. **이전 활동 과목명:** 텍스트 입력 (예: 통합과학)
3. **이전 탐구 주제:** 텍스트 입력 (예: 자율주행 자동차 라이다 센서의 한계점)
4. **세특 원문:** 넓은 Textarea
   * *안내문:* "생기부에 적힌 해당 과목의 세특 내용을 복사해서 그대로 붙여넣어 주세요."

### Section 2: ⏩ 타겟 활동 정보 (나의 목표)
학생이 이번 학기에 제출할 수행평가나 탐구 보고서의 목표를 적는 영역입니다.

1. **타겟 활동 시기:** `[학년 선택 (1~3)]` `[학기 선택 (1~2)]` (Dropdown)
2. **타겟 과목명:** 텍스트 입력 (예: 물리학Ⅰ)
   * **[핵심 기능 - 단원명 힌트 노출]**
     * 학생이 타겟 시기와 타겟 과목명을 입력하면, `curriculumData.ts`를 조회하여 해당 학기에 배우는 단원명들을 UI 하단에 가벼운 태그(Chips)나 텍스트로 미리 보여줍니다.
     * *예시:* "📘 이번 학기 물리학 수업에서는 이런 내용을 배워요! `#힘과 에너지` `#전기와 자기` `#빛과 물질`"
   * **[예외 처리 - 커리큘럼 폴백 전략]**
     * 개설 과목의 이름이 학교마다 조금씩 다를 수 있음을 감안합니다. 
     * 학생이 입력한 과목명이 `curriculumData.ts`에 존재하지 않을 경우: 단원명 힌트 UI를 노출하지 않고, AI 프롬프트 역시 사용자가 제시한 '과목명' 그 자체에 충실하게 반응하여 주제를 제안하도록 구축합니다.
3. **현재 고민이나 선생님의 지시사항 (옵션):** Textarea
   * *예시:* "물리 원리와 제 진로(컴공)를 엮어서 탐구보고서를 써야 합니다."

## 3. 데이터 매핑 로직 (Backend/Data)

`curriculumData.ts`의 월간 진도 데이터를 '학기' 단위로 묶어서 활용합니다.

* **1학기 (3월 ~ 7월):** 해당 학년/과목의 3, 4, 5, 6, 7월 데이터를 통합하여 배열로 반환.
* **2학기 (9월 ~ 12월):** 해당 학년/과목의 9, 10, 11, 12월 데이터를 통합하여 배열로 반환.
* AI에게 프롬프트 컨텍스트로 제공할 때는 중복을 제거한 단원명 리스트 포맷으로 전달합니다.

---

## 4. AI 에이전트 프롬프트 설계 방향 (Professor Agent)

프롬프트는 크게 3단계 사고 과정(Chain of Thought)을 따르도록 설계합니다.

1. **세특 원문 해독 (Fact & Competency Extraction):**
   * 입력된 세특에서 학생이 실제로 "어떤 구체적 행동(독서, 실험, 발표 등)"을 했고, "어떤 핵심 역량(분석력, 문제해결력 등)"을 인정받았는지 분리해서 파악합니다.
2. **타겟 과목 크로스 매핑 (Cross-over Mapping):**
   * 추출된 이전 역량/주제와 이번 학기 배울 '타겟 과목의 단원(Chapter)'을 충돌시켜 접점을 찾습니다. 단원명 데이터가 없을 경우 해당 과목의 보편적 수준에 기반하여 접점을 찾습니다.
3. **탐구 주제 산출 (Actionable Output):**
   * 대학이 선호하는 **심화(Level UP)**, **융합(Cross-over)**, **적용(Action Item)** 방향으로 3가지 이상의 구체적이고 뾰족한 탐구 주제 및 활동 가이드를 제안합니다. 진도 오류(할루시네이션)를 방지하기 위해 반드시 제공된 타겟 단원명 안에서 해결책을 찾도록 제한합니다.
