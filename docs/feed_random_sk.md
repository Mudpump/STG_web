# 탐구줍줍(Feed) 자동랜덤 생성 기능 로직

탐구줍줍(Feed)의 게시글 자동랜덤 생성 기능은 **4단계의 다중 AI 에이전트 시스템 체인(Multi-Agent Chain)**을 통해 짜임새 있게 생성되고 있습니다. 생성 프로세스의 진입점은 `FeedGenerator.tsx`이며, 여기서 설정한 `생성 개수`, `타겟 학년`, `타겟 시기` 등의 값을 넘겨받아 `runAgentChain`을 통해 각 단계별 에이전트가 역할을 수행합니다.

자세한 내부 로직은 다음과 같이 5단계로 분류할 수 있습니다.

### 1. Agent A: Topic Scout (주제 발굴)
가장 먼저 글의 '소재와 뼈대'를 기획하는 작업입니다.
- **분야 선정**: 기존에 생성된 전체 게시글 분포를 분석하여 **가장 게시글 수가 적은 카테고리(분야)를 우선적으로 탐색**합니다. 특정 학년과 카테고리에 맞는 '세부 교과목'과 '전공'을 매핑 데이터에서 무작위로 추출합니다.
- **접근 프레임워크 랜덤 선택**: 단순히 글을 쓰는 게 아니라, 생기부에 들어갈 법한 4가지 심도 있는 세특 기획 방식 중 하나를 무작위로 적용합니다.
  - `T1`: 교과서 개념을 논문 수준으로 파고드는 **심화 확장형**
  - `T2`: 최근 시사 이슈 등에 교과 관점을 녹이는 **이슈 연결형**
  - `T3`: 일상 속 속설 등을 학문적으로 검증해보는 **속설 검증형**
  - `T4`: 일상의 불편함을 지식으로 풀어내는 **일상 불편 해결형**
- 위 정보를 바탕으로 AI가 구체적인 "게시글 기획안 요약(Topic Data)"을 만들어냅니다.

### 2. 역할 캐스팅 (fetchCastFromDB)
게시글의 현실감을 높이기 위해, 주제 생성에 맞춰 글을 작성할 **[작성자]**와 이에 티키타카로 덧글을 달아 줄 **[조언자/댓글러] 페르소나들**(예: 팩트 체크 요원, 전략가 선배 등)을 DB(혹은 더미 데이터)에서 캐스팅합니다.

### 3. Agent B: Scenario Writer (대본 집필)
Agent A가 만든 기획안과 역할 캐스팅된 페르소나들을 바탕으로 실제 게시글과 댓글을 작성합니다.
- **게시글**: 작성자 페르소나에 이입하여 고등학생 특유의 고민이나 호기심이 담긴 어조로 텍스트를 작성합니다.
- **댓글 (티키타카)**: 캐스팅된 조언자들의 페르소나에 맞게 대화를 주고받습니다. 여기서 두루뭉술한 조언이 아닌 **"교과서에 나오는 구체적인 단원명, 개념어"**나 **"대학 학부 수준의 개론 전문 용어"**(예: 회귀분석, 마케팅 믹스 등)를 쓰도록 강력하게 프롬프팅되어 학문적 깊이를 부여합니다.

### 4. Agent M: Mentor (자기주도성 및 퀄리티 검수)
생성된 대본이 진짜로 학생에게 도움이 되고 신뢰할 수 있는지 '편집장' 역할을 하는 멘토 AI가 검수합니다.
- **검수 기준 4가지**:
  1. 교과 연계성 (진짜 수학/과학 개념을 썼는가?)
  2. 전공 적합성 (억지 연결이 아닌가?)
  3. 실현 가능성 (고등학생이 학교에서 할 수 있는 난이도인가?)
  4. 구체적 조언 (추상적이지 않고 구체적인 연구 방향이 나오는가?)
- **피드백 루프**: 검수 결과가 `REJECT(반려)`가 나올 경우, 멘토의 반려 사유(피드백)를 Agent B에게 다시 보내 **통과(PASS)할 때까지 대본을 수정/재작성**시킵니다.

### 5. Agent C: Scheduler (퍼블리싱 스케줄러)
멘토의 검수를 최종 통과한 시나리오를 받아 **실제 SNS 피드처럼 보이도록 데이터를 가공**합니다.
- 작성된 글과 스레드 형식의 댓글들을 취합합니다.
- 리얼리티를 살리기 위해 **무작위 뷰어 수(10~210회)**, **무작위 좋아요 수**, 댓글들의 **가상 작성 시간(예: n분 전)** 등을 계산하여 최종 `Post` 객체 데이터베이스 모델로 완성한 뒤 반환합니다.
